所存在问题：
1：信息注册用弹出窗口有些不妥，注册成功后需要弹出注册成功弹窗，双层弹窗不友好，切相关法规也需要时弹窗形式展开。
解决方案：1：依旧应用弹窗，但注册成功提示信息用改变按钮形态，即一个弹窗两用，相关法律法规用手风琴夹层形式展开。
					2：不应用弹窗，弹出新的页面（大部分网站注册皆是如此）。
2：注册信息明显不完善。需要给真实字段。以及各个字段的验证规则许给出。
3：患者医生的角色选择太过简单。一点验证没有如何保证医生信息真实度。
4：首页有系统消息栏，第二张图系统消息变为一级菜单式什么意思。
5：系统消息并非一直都有，应用弹出框方式。
解决方案：个人认为可以作为一级菜单，点击后，右侧展开。
6：复诊挂号为操作性按钮，其余为信息型按钮，前者为展开新功能，后者为展示各自信息，放在一起逻辑有一些问题。
解决方案：个人认为应把复诊挂号单提出来放在首页中作为一个单独的按钮。（好像人人里发表日志一定是在查看日志里面的一个单独按钮，而不是和查看日志同级）。
7：论坛本身就是一个系统，有自己的菜单以及消息提醒，是否真的嵌套当前系统中。
8：请给出患者首页详细字段，便于填充。
9：患者首页应有修改功能
